# Formating 방식

formating은 미리 정의된 문자열에 데이터 값을 끼워 넣어서 사람이 보기 좋은 문자열로 저장하는 과정

## '%'형식 문자열

'%' 형식화 연산자를 사용

템플릿에 끼워 넣을 값들을 연산자의 오른쪽에 % 뒤에 단일 값이나 tuple 형식으로 지정하는 방법

```python
a = 0b10111011
b = 0xc5f

print('이진수: %d, 십육진수: %d' %(a,b))
# 이진수: 187, 십육진수: 3167
```

C의 printf 함수에서 비록 되었다.
`%s`, `%x`, `%f` 등 형식 지정자를 지원한다.

### 단점

1. tuple 내 데이터 값의 순서 나 값의 타입을 바꾸면 타입 변환이 불가능하므로 오류가 발생

```python
key = 'my_var'
value = 1.234

# 실수로 순서를 바꿔 넣음
reordered_tuple = '%-10s = %.2f' %(value,key)
# TypeError : must be real number, not str

# 문자열의 순서가 바뀜
reoredered_string = '%.2f = %-10s' %(key,value)
# TypeError : must be real number, not str
```

- 이런 오류를 피하기 위해 순서를 계속 확인 해야함

2. 형식화를 하기 전 값을 살짝 변경해야 한다면 식의 가독성이 떨어진다.

```python
pantry = [
    ('아보카도', 1.25),
    ('바나나', 2.5),
    ('체리', 15),
]

for i, (item,count) in enumerate(pantry):
    print('#%d: %-10 = %d' %(
        i + 1,
        item.title(),
        round(count)
    ))
```

3. 문자열에서 같은 값을 여러 번 사용하고 싶다면 튜플에서 같은 값을 여러 번 반복해야 한다.

```python
template = '%s는 음식을 좋아해. %s가 요리하는 모습을 봐요'
name = '철수'

formatted = template %(name,name)
```

4. 형식화 식에 딕셔너리를 사용하면 문장이 번잡스러워 진다.
   - 단점 1번을 해결하기 위해 딕셔너리를 사용

## 내장함수 format과 str.format

파이썬 3부터는 C스타일 형식화 문자열 보다 format 내장 함수를 통해 모든 파이썬 값에 사용할 수 있다.

```python
a = 1234.5678
b = 'my 문자열'

print(format(a,',.2f'))
print('*',format(b,'^20s'),'*')
```

str타입에 format 메서드를 호출하여 사용 가능하다.
추가적으로 `:` 뒤에 있는 형식지정자를 format 메서드에 전달해 사용가능하다.

```python
key = 'my_var'
value = 1.234

formatted = '{} = {}'.format(key,value)
print(formatted)

formatted = '{:<10} = {:.2f}'.format(key,value)
print(formatted)
```

중괄호 안에 위치 인덱스를 전달 할 수도 있다.

### 단점

여전히 단점들은 남아 있는데, C형식의 formatting 방식에서의 2번째 단점인 `형식화를 하기 전 값을 살짝 변경해야 한다면 식의 가독성이 떨어진다.` 문제는 여전하다.

또한 단점 3,4번의 단점들도 여전히 존재 한다.  
`3. 문자열에서 같은 값을 여러 번 사용하고 싶다면 튜플에서 같은 값을 여러 번 반복해야 한다.`.  
`4. 형식화 식에 딕셔너리를 사용하면 문장이 번잡스러워 진다.`

## 인터폴레이션(interpolation)을 통한 형식 문자열 (f-문자열)

파이썬 3.6버전 이후 부터 사용 가능

```python
key = 'my_var'
value = 1.234

formatted = f'{key!r:<10s} = {value:.2f}'
print(formatted)
```

콜론 뒤에 형식 지정자를 사용할 수 있고,  
유니코드나 repr 문자열 변환 기능역시 사용 가능 하다.

f-문자열을 사용하면 위치 지정자 중괄호 안에 완전한 파이썬 식을 넣을 수 있다. 그렇기 때문에 위의 다른 Formating의 단점들도 해결이 가능하다.

```python
pantry = [
    ('아보카도', 1.25),
    ('바나나', 2.5),
    ('체리', 15),
]

for i, (item,count) in enumerate(pantry):
    print(f'#{i+1}: '
    f'{item.title():<10s} = ')
    f'{round(count)}'
```

f-문자열이 제공하는 표현력, 간결성, 명확성을 고려할 때 파이썬 프로그래머가 사용할 수 있는 Formating 옵션 중 f-문자열이 최고다.

# 복잡한 식을 쓰는 대신 도우미 함수를 작성하라.

파이썬은 문법이 간결하여, 상당한 로직이 들어가는 식도 한 줄로 매우 쉽게 작성할 수 있다.

```python
from urllib.parse import parse_qs

my_values = parse_qs('빨강=5&파랑=0&초록=',
                    keep_blank_values=True)

print(repr(my_values))
# {"빨강":['5'], "파랑":['0'], "초록":['']}
```

파라미터가 없거나 비어 있는 경우 0이 디폴트 값으로 대입 되면 좋을 것이다.

```python
green = int(my_values.get('초록',[''])[0] or 0)
```

이런 식으로 사용 가능하다. 하지만 이 코드는 바로 어떤 역할을 하는지 알기 어렵다.
파이썬은 코드를 간결하게 유지하면서 명확하게 표현할 수 있는 if/else 조건식이 있다.

```python
green_str = my_values.get('초록',[''])
green = int(green_str[0] if green_str[0] else 0)
```

가독성은 올라갔다. 하지만 red, blue, green 에 모두 이런 코드를 작성한다면 불필요한 코드를 작성해야 한다.

이럴 때 도우미 함수를 작성하여, 사용하면 코드는 짧아지고 의미를 알기 위해서는 함수만 보면 된다.

```python
def get_first_int(values,key,default = 0)
    value_str = values.get(key,[''])
    return int(value_str[0] if value_str[0] else 0)


green = get_first_int(my_values, '초록')
```

함수에 식이 더 복잡해 진다면, 더 작은 조각으로 나눠서 로직을 도우미 함수로 옮길지 고려하면 된다.

무조건 짧게 줄여 쓰는 것 보다는 가독성을 좋게 하는 것이 더 가치 있다.  
복잡한 식을 표현할 수 있는 파이썬의 함축적인 문법이 이런 지저분한 코드를 만들어 내지 않도록 해라.

# 인덱스 대신 대입을 사용해 언패킹을 해라

마트의 장부가 있다고 하자.

```python
snack_bar = {
    '감자칩':50,
    '빼빼로':10,
    '짱구':20,
}

items = tuple(snack_bar.items())
print(items)
```

인덱스 방식과 언패킹 방식을 보면

```python
# 인덱스 방식
potato_chip = items[0] #('감자칩', 50)
product = potato_chip[0]
counts = potato_chip[1]
print(product,counts)

# 언패킹 방식
product, counts = potato_chip
print(product,counts)
```

언패킹은 인덱스를 사용하는 것보다 시각적인 잡음이 적다. iterable 안에 여러 계층으로 이터러블이 들어간 경우 등 다양한 패턴을 언패킹 구문에서 사용할 수 있다.

특히 큰 차이점을 보이는 건, 리스트의 두 원소의 위치를 변경 할 때 인데, 인덱스는 임시 변수를 사용하지만, 언패킹은 임시변수를 사용하지 않는다.

```python
def bubble_sort(a):
    for _ in range(len(a)):
        for i in range(1,len(a)):
            if a[i] < a[i-1]:
                # 인덱스 방식
                temp = a[i]
                a[i] = a[i-1]
                a[i-1] = temp

                # 언패킹 방식
                a[i-1],a[i] = a[i],a[i-1]
```

왜 언패킹 방식이 가능한지 보자.

1. 대입문의 우항이 계산 되고, 그 결과값이 **이름이 없는 새로운 tuple에 저장된다.**
2. 그 후 대입문의 좌항에 있는 언패킹 패턴을 통해 임시 tuple에 있는 값이 각각 변수에 저장된다.
3. 그러면서 위치가 변경이 되고, **마지막으로 이름이 없는 tuple은 조용히 사라진다.**

또한 비슷한 사례로 for 루프 또는 그와 비슷한 다른 요소(컴프리헨션,제너레이터 식)에서 언패킹을 사용할 수 있고, 인덱스 보다 코드가 짧아 진다.

결과적으로 언패킹 방식은 인덱싱 방식 보다. 파이썬 다운 방식이며, 가독성이 좋고 이해하기도 쉽다.

파이썬에서는 list, 함수의 인자, 함수의 반환 값, 키워드 인자 등에 언패킹 기능을 제공한다. 이런 기능들은 다른 chapter에서 조금씩 다룬다.

# range보다는 enumerate를 사용하라.

range 내장 함수는 어떤 정수 집합을 이터레이션하는 루프가 필요할 때 유용하다.

```python
from random import randint

for i in range(32):
    if randint(0,1):
        random_bits |= 1 << i

```

리스트를 이터레이션하면서 리스트의 몇 번째 원소를 처리 중인지 알아야 할 때가 있다.
이렇게 순서를 알아야 할 때 range 보다는 enumerate 사용이 더 좋다.

```python
flavor_list = ['바닐라','초콜릿','딸기']

# range
for i in range(len(flavor_list)):
    flavor = flavor_list[i]
    print(f'{i+1}: {flavor}')

# enumerate
for i,v in enumerate(flavor_list):
    print(f'{i+1}: {v}')
```

이렇게 코드가 훨씬 깔끔하다.  
심지어, enumerate는 어디서부터 수를 세기 시작할지 지정할 수도 있다.

```python
for i,v in enumerate(flavor_list,1):
    print(f'{i}: {v}')
```

또한 enumerate 는 이터레이터를 lazy generator(지연 계산 제너레이터)로 감싼다.

```python
it = enumerate(flavor_list)
print(next(it)) # (0, '바닐라')
print(next(it)) # (1, '초콜릿')

```

# 여러 이터레이터에 대해 나란히 루프를 수행하려면 zip을 사용하라.

아래 두개의 리스트가 있다고 했을 때  
두 리스트를 동시에 사용하여, 이름의 길이를 활용해 제일 긴 이름을 찾는 반복문을 실행해 보자.

```python
names = ["Cecilia", "남궁민수", "손민수"]
counts = [len(n) for n in names]
```

names 소스 리스트의 길이를 사용해 이터레이션할 수 있다.

```python
longest_name = None
max_count = 0

for i,v in enumerate(names):
    count = counts[i]
    if count > max_count:
        longest_name = v
        max_count = count
print(longest_name)
```

range보다는 깔끔하지만, 여전히 두 리스트를 동시에 사용하다 보니, 깔끔하지 않다.
이럴때 zip 은 둘 이상의 이터레이터를 지연 계산 제너레이터를 사용해 묶어준다.
제너레이터는 각 이터레이터의 다음 값이 들어 있는 튜플을 반환한다. 이 튜플을 for 문에서 바로 언패킹할 수 있다.

```python
for name, count in zip(names,counts):
    if count > max_count:
        longest_name = name
        max_count = count
```

zip은 자신이 감싼 이터레이터 원소를 하나씩 소비한다. 따라서 메모리를 다 소모해서 프로그램이 중단되는 위험 없이 아주 긴 입력도 처리할 수 있다.

## 길이가 서로 다른 이터레이터

zip은 자신이 감싼 이터레이터 중 어느 하나가 끝날 때까지 튜플을 내놓는다.
따라서, 출력은 가장 짧은 입력의 길이와 같다. 물론 길이가 모두 같다면 이 방식이 잘 작동한다.

하지만 긴 이터레이터의 뒷부분을 버리는 zip 기능이 바람직하지 못한 경우도 있다.
그 때 사용하는 것이 `itertools` 내장 모듈에 들어있는 `zip_longest`이다.

zip_longest는 존재하지 않는 값을 자신에게 전달된 fillvalue로 대신한다. default fillvalue는 None이다.

```python
import itertools
names.append("Roslind")

for name, count in itertools.zip_longest(names,counts):
    print(f'{name}:{count}')

# Cecilia:7
# 남궁민수:4
# 손민수:3
# Roslind:None
```

# 대입식을 사용해 반복을 피하라. (왈러스 연산자 (:=))

파이썬 3.8에서 적용된 구문

```python
fresh_fruit ={
    '사과':10,
    '레몬':5,
    '바나나':8,
}

def make_lemonade(count):
    ...

def out_of_stock():
    ...

count = fresh_fruit.get('레몬',0)
if count:
    make_lemonade(count)
else:
    out_of_stock()

```

레모네이드를 주문하였다면 과즙을 낼 레몬이 과일 바구니에 최소 하나 있어야 하고,  
레몬의 개수를 읽어서 그 값이 0이 아닌지 검사하는 코드다.

count 변수는 if 문의 첫번째 블록 안에서만 쓰인다.  
if 앞에서 count를 정의하면 else블록이나 그 이후의 코드에서 count변수에
접근해야 할 필요가 있는 것처럼 보이기 때문에 실제보다 변수가 중요해 보인다.

파이썬에서는 이런 식으로 값을 가져와서 그 값이 0이 아닌지 검사한 후 사용하는 패턴이 자주 발생한다.

이럴때 왈러스 연산자를 사용해서, count 변수가 if 문 안에서 쓰이는 것처럼 처리할 수 있다.

```python
if count := fresh_fruit.get('레몬',0):
    make_lemonade(count)
else:
    out_of_stock()
```

---

또한 조건에 따라 현재 위치를 둘러싸는 영역에 있는 변수에 값을 대입하고 그 변수를 바로 함수 호출에 사용하는 경우를 들 수 있다.

```python

pieces = 0
count = fresh_fruit.get('바나나',0)

if count >=2:
    pieces = slice_banana(count)

try:
    smoothies = make_smoothies(pieces)
except OutOfBananas:
    out_of_stock()

# 또 다른 방법
count = fresh_fruit.get('바나나',0)

if count >=2:
    pieces = slice_banana(count)
else:
    pieces = 0
try:
    smoothies = make_smoothies(pieces)
except OutOfBananas:
    out_of_stock()

# 왈려스 연산자
pieces = 0
if (count := fresh_fruit.get('바나나',0)) >2:
    pieces = slice_banana(count)

try:
    smoothies = make_smoothies(pieces)
except OutOfBananas:
    out_of_stock()
```

보통 pieces = 0 대입을 먼저 하는 첫번째 방식을 많이 선호 한다.  
두번째 방법도 틀린건 아니지만, pieces가 두 위치에서 정의되는 것 처럼 보여 조금 이상하게 느껴질 수 있다.  
왈러스 연산자는 count 정의가 if 문 앞에 없기 때문에 count 변수는 더 이상 강조되지 않고 pieces 변수를 따라가기 쉬워진다.

들여쓰기와 내포가 줄어서 가독성은 훨씬 좋아진다.
이 방식은 while 문에서도 사용 가능한데,

```python
bottles = []
while True:
    fresh_fruit = pick_fruit()
    if not fresh_fruit:
        break

    for fruit, count in fresh_fruit.items():
```
