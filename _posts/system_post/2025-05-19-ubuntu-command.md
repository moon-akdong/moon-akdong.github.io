# 파일 조회 및 권한

## 파일 확인

디렉토리 안에서, 파일과 파일 권한을 확인

- `-al` : 숨김 파일도 확인 가능

```
$ ls -al
```

![alt text](/assets/post/ubuntu/file_list.png)

- 맨 앞에 `drwxr-xr-x@`는 파일의 권한을 나타낸다.
  - `d` : 폴더 `-`은 파일
  - `r` : 읽기
  - `w` : 쓰기
  - `x` : 실행

그룹을 나눌 수 있는데, 맨 처음 글자는 폴더 or 파일을 나타내고,  
그 뒤에 3개씩 나누어 소유자 권한, 소유자가 속한 그룸에 대한 권한, 모든 사용자들에 대한 권한

예시로 `drwxrwxr-x`라면

- `drwx` : 첫번째 그룹 소유자는 모든 권한을 행상 할 수 있고, 폴더 이다.
- `rwx` : 두번째 그룹 소유자가 속해 있는 그룹은 읽기,쓰기, 실행 다 할 수 있음
- `r-x` : 세번째 그룹, 모든 사용자는 읽고 실행하기만 가능하다.

## 파일 권한 변경

```
$ chmod [권한 설정] [파일 명]

chmod 755 script.sh   # 파일에 적용
chmod -R 755 /var/www  # 디렉토리 전체에 재귀 적용

```

| 명령어      | 소유자 | 그룹 | 기타 | 주요 용도                      |
| ----------- | ------ | ---- | ---- | ------------------------------ |
| `chmod 777` | rwx    | rwx  | rwx  | 모두 전체 권한 (보안상 비권장) |
| `chmod 755` | rwx    | r-x  | r-x  | 일반적인 실행 파일/디렉토리    |
| `chmod 644` | rw-    | r--  | r--  | 일반적인 텍스트/설정 파일      |
| `chmod 600` | rw-    | ---  | ---  | 개인 키, 민감한 파일           |
| `chmod 700` | rwx    | ---  | ---  | 개인 스크립트                  |
| `chmod 400` | r--    | ---  | ---  | 읽기 전용 보호 파일            |

# 프로세스

## 상태 확인

```
$ ps
$ ps aux

```

추가적으로 grep과 함께 사용도 가능하다.
`ps aux | grep 프로세스 명` : 프로세스가 실행 중인지를 확인하고, 관련 프로세스에 대한 정보 출력

- `a` : 시스템을 사용하는 모든 사용자의 프로세스 출력
- `u` : 프로세스 소유자에 대한 상세 정보 출력
- `x` : 터미널에 로그인한 후 실행한 프로세스가 아닌 프로세스들도 조회, 주로 데몬 프로세스까지 확인하기 위해 사용 ps 명령은 현재 쉘 에서 실행한 프로세스들만 보여주기 때문에 이 옵션을 사용하는 경우가 많음

![alt text](/assets/post/ubuntu/ps_aux.png)

> 데몬 프로세스(Daemon Process)
> 백그라운드에서 조용히 실행되는 프로세스다. 사용자가 직접 실행하거나 터미널에서 제어하는 일반 프로세스와 달리, 데몬은 시스템이 시작될 때 자동으로 실행되어 종료될 때까지 계속 돌아간다.  
> **Daemon Process 예시**
> |데몬|역할|
> |---|---|
> |`sshd`|SSH 연결 처리|
> |`nginx`|웹 서버|
> |`cron`|스케줄 작업 실행|
> |`systemd`|시스템 초기화 및 서비스 관리|

## 프로세스 종료

`ps aux` 명령어 시 나오는 결과
![alt text](/assets/post/ubuntu/ps_aux.png)

여기서 PID를 이용해 프로세스를 종료할 수 있다.

```
kill [보낼 신호] [종료할 프로세스의 PID]

kill 1234 # 정상 종료 시도
kill -9 1234 # 그래도 안되면 강제 종료
```

| 신호      | 번호 | 의미                                           |
| --------- | ---- | ---------------------------------------------- |
| `SIGTERM` | 15   | 정상 종료 요청. 프로세스가 정리 후 스스로 종료 |
| `SIGKILL` | 9    | 강제 종료. 프로세스가 거부할 수 없음           |
| `SIGHUP`  | 1    | 재시작 요청. 설정 파일 다시 읽기 등에 사용     |
