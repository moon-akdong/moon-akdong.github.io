# AlexNet

[AlexNet](https://proceedings.neurips.cc/paper_files/paper/2012/file/c399862d3b9d6b76c8436e924a68c45b-Paper.pdf)

ImageNet 2012에서 top-5 error 15.3% 달성 하면서 2위와 압도적 격차로 딥러닝 시대를 열었다는 의의를 가진다.

## Dataset

ImageNet은 22,000개의 카테고리, 1,500만장 이상의 레이블된 고해상도 이미지 데이터셋이다.

AlexNet에서 ILSVRC 라는 대회에서 ImageNet 서브셋을 사용해서 좋은 결과를 얻었는데, 이 서브셋은 120만장의 학습 이미지, 5만장의 검증 이미지, 15만장의 테스트 이미지 데이터가 있다.

논문에서는 224x224(실제 계산으로는 227x227)로 고정 해상도로 다운 샘플링을 하였다.

### 전처리

#### 리사이징

직사각형 이미지의 경우, 먼저 짧은 쪽이 256이 되도록 이미지를 리스케일한 후 중앙의 256x256 패치를 잘라낸다.

#### 픽셀

전체 이미지의 각 채널의 모든 픽셀에 대해 각 픽셀의 평균값을 뺐다.
즉 픽셀의 중앙화된 원시 RGB 값으로 네트워크를 학습

- 평균 빼기(Mean Subtraction) : 데이터를 0 중심으로 이동 (Centering)
- 정규화(Normalization) : 평균 빼고, 표준편차로 나누기 하면 정규화

당시에는 평균 빼기만으로도 충분히 효과적으로 생각했다

### Data Augmentation

#### 이미지 패치와 수평 반점

256x256에서 되도록 이미지 리사이징을 하고, 중앙의 227x227 패치를 잘라냈지만, 데이터 수량을 늘리기 위해 이미지에서 랜덤으로 227x227 패치를 추출(+수평 반전)하여 네트워크를 학습시킨다. 이는 학습 세트의 크기를 2048배 증가시킨다.

테스트 시에는 5개의 227x227패치(4개의 코너 패치와 중앙 패치) 및 그 수평 반전(10개)

> 4개의 코너와 중앙 패치 : 5개
> 5개의 수평 반전 \*2 : 10개

10개의 테스트 데이터의 결과를 합쳐 평균으로 결과를 예측한다. TTA(Test Time Augmentation)라고 하는데, 노이즈가 줄고 더 안정적인 예측을 할 수 있음, 앙상블의 원리와 유사한 기법

#### RGB 채널에 PCA기반 색상 노이즈 추가

RGB 각 채널 별 픽셀 값들을 합쳐서 N개의 [R,G,B]벡터의 3x3 공분산 행렬 계산 -> PCA
이렇게 나온 아래값들

- **p (고유벡터)** → 주성분 **방향**
- **λ (고유값)** → 그 방향의 **분산 크기**

p1 방향으로 데이터가 λ1만큼 퍼져있다
p2 방향으로 데이터가 λ2만큼 퍼져있다
p3 방향으로 데이터가 λ3만큼 퍼져있다

이렇게 나온 PCA값들과 가우시안 분포에서 α 값을 추출

$$Ixy = [IR_xy, IG_xy, IB_xy]ᵀ$$

```
IR_xy : (x,y) 위치의 R 채널 값
IG_xy : (x,y) 위치의 G 채널 값
IB_xy : (x,y) 위치의 B 채널 값
```

$$Ixy = Ixy + [p1,p2,p3][α1λ1, α2λ2, α3λ3]ᵀ$$

```
기존 픽셀값 + PCA 기반 색상 노이즈
```

## Model Architecture

### CNN

1. ImageNet만큼 수 많은 데이터셋을 학습 시키려면 capacity가 큰 모델이 필요하다.
2. ImageNet 같은 대규모 데이터셋도 일상생활에서의 모든 문제를 완전히 커버를 못하기 때문에, 데이터에 없는 모든 데이터를 보완하기 위해 많은 사전 지식을 가지고 있어야 한다.(Prior Knowledge)

위 두 가지 이유로 Convolution Nueral Network(CNN)의 사용이 필요하다.

CNN의 용량은 깊이, 너비(depth and breadth)를 조절함으로써 제어할 수 있으며, 이미지의 특성(통계적 정상성과 픽셀 의존성의 지역성 - namely, stationarity of statistics and locality of pixel dependencies)에 강하고 대체로 올바른 가정을 내포하고 있다.

> - stationarity of statistics (통계적 정상성)
>
> * 이미지의 어떤 위치에서든 동일한 특징 검출기가 유효하다는 가정입니다. (가중치 공유 -> 파라미터 절약)
>
> - locality of pixel dependencies (픽셀 의존성의 지역성)
>
> * 가까운 픽셀끼리만 강하게 연관되어 있다는 가정입니다.(로컬 receptive field -> 합성곱 구조)

전통적인 feedforward nueral network 에 비해 연결 수와 파라미터가 훨씬 적어 학습이 쉬우며 이론적 최고 성능도 약간만 낮다.

최종 네트워크는 5개의 CNN Layer와 3개의 Fully Connected layer 로 이루어져 있다.

당시에는 GPU(580 3gb)에서 사용 가능한 VRAM 크기가 작았기 때문에 CNN Filter를 반으로 나누고, 아래 이미지와 같은 형태의 Architecture가 나온다.

Layer3에서 두 GPU의 모든 커널 맵에서 입력 받음
Layer4~5에서는 같은 GPU의 커널 맵에서만 입력 받음

### ReLU

기존에 사용하던 비선형 함수인 tanh, sigmoid와 같은 포화함수(Saturating Function)

> 포화함수(Saturating Function)
> 입력 값이 매우 크거나 작아지면, 출력이 그 범위의 끝에 포화 되어 버려서 기울기가 0에 가까워 진다.
> Gradient Vanishing 문제가 발생할 수 있다.

$$Sigmoid : f(x) = (1 + e⁻ˣ)⁻¹$$

$$Tanh : f(x) = tanh(x)$$

그렇기 때문에 AlexNet에서는 비포화 비선형 함수인 ReLU(Rectified Linear Units)라고 불리는 비선형 네트워크를 사용한다.

$$ReLU:f(x)=max(0,x)$$

Dying ReLU 문제 : x ≤ 0이면 기울기가 항상 0이라, 한번 비활성화된 뉴런이 영영 죽어버리는 문제가 발생하기 때문에,
이후에는 Leaky ReLU, PReLU등이 이를 보완하기 위해 나왔다.

```
Leaky ReLU — Maas et al. (2013), "Rectifier Nonlinearities Improve Neural Network Acoustic Models"
PReLU — He et al. (2015), "Delving Deep into Rectifiers" (ResNet 저자)

```

일부 연구에서 `|tanh(x)|`가 contrast normalization(대비 정규화) + local average pooling 조합에서 잘 작동했지만, 그 데이터셋(Caltech-101)에서는 과적합 방지가 주된 관심사였고, ReLU가 보여주는 학습 가속 효과와는 다른 맥락이다.

$$f(x) = |tanh(x)|$$

### Local Response Normalization

ReLU는 포화를 방지하기 위한 입력 정규화가 필요 없다는 바람직한 특성을 가지고 있다고 한다.
다음의 LRN 방식이 일반화에 도움이 되는 것을 발견했다.

$$b^i_{x,y} = a^i_{x,y} \Bigg/ \left( k + \alpha \sum_{j=\max(0,\, i-n/2)}^{\min(N-1,\, i+n/2)} \left(a^j_{x,y}\right)^2 \right)^\beta$$

- $i$ : kernel(chennal)
- $a^i_{x,y}$ : i번째 커널(필터)의 (x, y) 위치에서 ReLU 적용 후의 출력값
- $n$ : 정규화에 사용할 인접 커널 수 (하이퍼파라미터)
- $b^i_{x,y}$ : LRN 적용 후 정규화된 출력값
- $N$ 해당 레이어의 전체 커널 수
- $k$, $α$, $$β$ : 하이퍼파라미터 (AlexNet에서 $k=2$, $α$=0.0001$, $β=0.75$ 사용)

직관적으로 보면 i번째 커널 주변 n개의 커널 출력값들을 제곱합산해서 나눠주는 것, 강하게 반응한 뉴런 주변을 억제하는 측면 억제(lateral inhibition) 효과입니다.

LRN 을 적용했을 때 top-1 1.4%, top-5 1.2% 개선을 했다.
CIFAR-10에서는 13% -> 11%로 개선

### MaxPooling (Overlapping Pooling)

이전에는 Pooling 을 겹치지 않게 사용했다. s = z로 설정(s는 픽셀 간격, z는 pooling size) 이렇게 겹치지 않았다.

Overlapping은 s=2, z=3 이 방식을 사용한다.
top-1,-5 에서 0.4%, 0.3% 개선이 되었음(Non-Overlapping 과의 비교인거 같음)

## Overfitting

- Data Augmentation에서 다루었던 내용도 Overfitting을 막기위해 한 시도였다.

### Dropout

hidden neuron의 출력은 0.5확률로 0으로 설정 하는 기능이다.
hidden neuron의 출력을 0.5의 확률로 0으로 설정하는 것으로 구성됩니다. 이렇게 "dropout"된 뉴런은 순전파에 기여하지 않으며 역전파에도 참여하지 않습니다.

다른 뉴런들의 다양한 랜덤 부분집합과 함께 유용한 더 강건한 특징을 학습하도록 강제됩니다.

수럼 속도는 2배 늘어난다.

## Details of learning

배저희는 배치 크기 128, 모멘텀 0.9, weight decay 0.0005로 확률적 경사 하강법(SGD)

이 작은 양의 weight decay가 모델 학습에 중요하다는 것을 발견했습니다.

weight decay는 단순한 정규화 기법이 아닙니다. 모델의 학습 오류 자체를 줄여줍니다. 가중치 w에 대한 업데이트 규칙은 다음과 같습니다.

$$v_{i+1} := 0.9 \cdot v_i - 0.0005 \cdot \epsilon \cdot w_i - \epsilon \cdot \left\langle \frac{\partial L}{\partial w} \bigg|_{w_i} \right\rangle_{D_i}$$

$$w_{i+1} := w_i + v_{i+1}$$

- $i$ : iteration 인덱스
- $v$ : 모멘텀 변수
- $\epsilon$ : 학습률 (learning rate)
- $w$ : 가중치
- $D_i$ : i번째 미니배치

- $\left\langle \frac{\partial L}{\partial w} \bigg\|_{w_i} \right\rangle_{D_i}$ : 번째 배치에서 $w_i$로 평가한 목적함수의 기울기 평균

weight_decay는 L2 Regularization과 같은 개념
모멘텀 변수 v : 이전 업데이트 방향과 크기를 기억하는 변수입니다.

### 순수 SGD

$$w_{i+1} := w_i - \epsilon \cdot \nabla L$$

### AlexNet SGD (모멘텀 + weight decay)

$$v_{i+1} := 0.9 \cdot v_i - 0.0005 \cdot \epsilon \cdot w_i - \epsilon \cdot \left\langle \frac{\partial L}{\partial w} \bigg|_{w_i} \right\rangle_{D_i}$$

$$w_{i+1} := w_i + v_{i+1}$$
